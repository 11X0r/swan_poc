Swan CORS Security Vulnerability

Impact Demonstration

This page demonstrates a critical security vulnerability in Swan's banking API. It allows an attacker to:

  1. Circumvent the browser's Same-Origin Policy (fundamental web security mechanism)
  2. Access sensitive user banking data without permission
  3. Perform unauthorized banking operations as the victim
  4. All of this is done silently when a user simply visits a malicious page

Note to Swan Security Team: This vulnerability cannot be exploited without CORS. Your statement that "an attacker could simply use this GraphQL mutation without exploiting CORS" is incorrect due to the Same-Origin Policy.
